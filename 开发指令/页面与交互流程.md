## 1. 首页 `pages/home/index`

| 项 | 内容 |
|---|---|
| 接口 | `listCategories`（展示部分热门品类，可选） |
| 要素 | 顶部 banner；两个主入口按钮：**【按品类预约】**→ category，**【拍照提交】**→ order-create?source=photo；「我的订单」快捷入口 → order-list |

## 2. 品类列表 `pages/category/index`

| 项 | 内容 |
|---|---|
| 接口 | `listCategories` |
| 要素 | 品类卡片列表（图标 + name + priceRef + unit）；可多选品类；底部「下一步」 |
| 交互 | 选中品类后 → order-create?source=category，把选中品类带过去（可经 URL 传 id 串，或用全局/缓存暂存所选） |
| 空态 | 无品类时显示"暂无可回收品类" |
| 规则 | **支持多选品类**（PRD 5.2）；本页只选品类，预估重量/件数在下单页填 |

## 3. 提交订单 `pages/order-create/index`

最核心的页面。根据 `source` 渲染不同表单。

| 项 | 内容 |
|---|---|
| 入参(query) | `source=category`（带所选品类）或 `source=photo` |
| 接口 | 进入时 `getAddressList`（取默认地址）；提交时 `createOrder`；拍照时 `wx.chooseMedia`+`wx.cloud.uploadFile` |

### 表单要素
| 区块 | source=category | source=photo |
|---|---|---|
| 物品 | 所选品类列表，每项填**预估重量/件数**（按品类需满足最低起收量） | **拍照/相册选图**（最多 9 张），可加文字描述 |
| 地址 | 显示已选地址，点击 → address/list 选择 | 同左 |
| 预约时间 | 选择**日期 `appointDate`** + **时间段 `appointSlot`**（时间段如 09:00-12:00 / 12:00-18:00；范围不限） | 同左 |
| 备注 | 选填文本框 | 同左 |
| 提交 | 校验必填 → `createOrder` | 同左 |

### 拍照上传流程（source=photo）
```
wx.chooseMedia 选图
  → 逐张 wx.cloud.uploadFile(cloudPath: recycle/{openid}/{ts}.jpg)
  → 收集返回的 fileID
  → createOrder 时把 fileID 数组放进 photos
```
> **图片孤儿问题（R12，第一版已知权衡）：** 图片是"先上传、后提交订单"，若用户传了图却没提交（或中途退出），这些 fileID 不会进任何订单，成为云存储里的"孤儿文件"。第一版**接受少量孤儿**，不做复杂的事务清理；治理方式：① 路径统一前缀 `recycle/{openid}/` 便于识别；② 后期用定时云函数（或管理后台）扫描清理"未被任何订单引用"的文件。前端可做的小优化：删除已选图时同步调用 `wx.cloud.deleteFile` 删掉已上传的那张。

### 提交结果
- 成功：`wx.showToast("提交成功")` → `wx.redirectTo` 到 order-detail?id=新订单id。
- 失败：按错误码 toast（如未选地址 → "请先选择地址"）。

### 异常/边界
| 场景 | 处理 |
|---|---|
| 未填地址 | 禁止提交，提示选地址 |
| 未选品类 / 未拍照 | 禁止提交 |
| 重量为 0 或非数字 | 校验提示 |
| 按品类未达最低起收量 | 提交返回 `BELOW_MIN_QUANTITY`，提示"未达最低起收量" |

## 4. 地址列表 `pages/address/list`

| 项 | 内容 |
|---|---|
| 接口 | `getAddressList`；删除调 `deleteAddress` |
| 要素 | 地址卡片（联系人/电话/地址 + 默认标签）；每项可编辑/删除；底部「新增地址」 |
| 双模式 | ①管理模式（从个人中心进）；②选择模式（从下单页进，点选后回填并返回 order-create） |
| 跳转 | 新增/编辑 → address/edit |
| 空态 | "还没有地址，去新增" |

## 5. 地址编辑 `pages/address/edit`

| 项 | 内容 |
|---|---|
| 入参 | `id`（编辑时带，新增时不带） |
| 接口 | `saveAddress` |
| 要素 | 联系人、手机号、省市区（可用微信 `chooseAddress` 或地区选择器）、详细地址、设为默认开关 |
| 校验 | 必填项 + 手机号格式 |
| 完成 | 保存成功返回上一页 |

---

## 6. 我的订单 `pages/order-list/index`

| 项 | 内容 |
|---|---|
| 接口 | `getOrderList`（带 `statusGroup` 筛选 + `page`/`pageSize` 分页，返回 `{list,total,hasMore}`） |
| 要素 | 顶部状态筛选 tab（全部/进行中/已结束）；订单卡片：orderNo、summary、状态文案、预约日期+时间段 |
| 交互 | 点卡片 → order-detail?id=xxx；列表**触底加载下一页**（hasMore 为 true 时 page+1） |
| 空态 | "暂无订单，去首页预约" |
| tab→statusGroup 映射 | 全部=`all`；进行中=`ongoing`(submitted,confirmed,assigned,recycling)；已结束=`done`(completed,canceled,rejected) |

---

## 7. 订单详情 `pages/order-detail/index`

| 项 | 内容 |
|---|---|
| 入参 | `id` |
| 接口 | `getOrderDetail`（含 photoUrls）；可取消时调 `cancelOrder` |
| 要素 | **状态进度条**（按数据字典 3.3 文案/进度）；物品信息（品类+重量 或 图片九宫格）；地址快照；预约日期+时间段；备注；估价 estimatePrice；完成后显示 finalWeight/finalPrice |
| 操作 | `submitted/confirmed`（上门前）显示「取消订单」按钮；rejected 显示不可回收原因 |
| 异常 | `ORDER_NOT_FOUND` → 提示并返回 |

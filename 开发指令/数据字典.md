# 云回收 · 数据字典

> 配套：[云回收项目PRD.md](./云回收项目PRD.md) · [云回收技术路线.md](./云回收技术路线.md) · [云函数接口契约.md](./云函数接口契约.md) · [页面与交互流程.md](./页面与交互流程.md)
> 对应 Java 开发者认知：本文 ≈ 建表语句 + 字段注释 + 权限说明。
> 注意：云开发是**文档数据库（NoSQL）**，"集合 ≈ 表，文档 ≈ 行（自由 JSON）"，字段不需预先建。本文是约定，不是强制 schema。

## 0. 通用约定

> **读写模型（最重要，先读）：** 本项目**前端一律不直连数据库**，所有读写都经云函数 `quickstartFunctions`。由此引出两条铁律：
> 1. **`_openid` 必须手动写**：云函数是管理员身份，写入数据时不会自动带上调用者 openid。凡需归属用户的数据（addresses/orders），必须在云函数里 `data._openid = cloud.getWXContext().OPENID` 手动赋值。
> 2. **集合权限收紧为"所有用户不可读写"**：安全边界落在云函数，而非数据库权限。用户数据隔离靠云函数内 `where({ _openid: OPENID })` 实现。

| 项 | 说明 |
|---|---|
| `_id` | 每个文档主键，不传则系统自动生成字符串 |
| `_openid` | 归属用户标识。**⚠️ 本项目全部走云函数读写，`_openid` 不会自动写入，必须在云函数里手动赋值**：`data._openid = cloud.getWXContext().OPENID`。（"自动写入"仅发生在小程序端 `wx.cloud.database()` 直连数据库时；云函数是管理员身份，不带调用者身份） |
| 时间字段 | 统一存毫秒时间戳（Number）或 `db.serverDate()`，避免客户端时间不准 |
| 命名 | 集合名小写，字段名小驼峰 |

## 1. 集合 `categories`（可回收品类）

平台维护，用户只读。

| 字段 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| `_id` | string | 自动 | - | 主键 |
| `name` | string | 是 | - | 品类名，如"纸箱/废纸" |
| `unit` | string | 是 | - | 计量单位：`kg` / `件` |
| `priceRef` | string | 否 | "" | 参考价文案，如"0.8元/kg"（仅展示，不参与计算） |
| `icon` | string | 否 | "" | 品类图标 fileID |
| `sortOrder` | number | 否 | 0 | 列表排序，升序 |
| `enabled` | boolean | 否 | true | 是否上架，false 不展示给用户 |

### 示例数据（待确认，开发可先用）
```jsonc
[
  { "name":"纸箱/废纸", "unit":"kg", "priceRef":"0.8元/kg", "sortOrder":1, "enabled":true },
  { "name":"塑料瓶/塑料", "unit":"kg", "priceRef":"0.5元/kg", "sortOrder":2, "enabled":true },
  { "name":"易拉罐/金属", "unit":"kg", "priceRef":"2.0元/kg", "sortOrder":3, "enabled":true },
  { "name":"旧衣物", "unit":"kg", "priceRef":"0.2元/kg", "sortOrder":4, "enabled":true },
  { "name":"旧家电", "unit":"件", "priceRef":"面议", "sortOrder":5, "enabled":true }
]
```

### 权限设置（控制台）
**所有用户可读，仅管理员可写**。

## 2. 集合 `addresses`（用户地址）

| 字段 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| `_id` | string | 自动 | - | 主键 |
| `_openid` | string | **云函数手动写入** | - | 归属用户，下单/存地址时由云函数 `getWXContext().OPENID` 赋值 |
| `contactName` | string | 是 | - | 联系人 |
| `phone` | string | 是 | - | 手机号 |
| `region` | string | 是 | - | 省市区，如"上海市浦东新区" |
| `detail` | string | 是 | - | 详细地址（门牌等） |
| `isDefault` | boolean | 否 | false | 是否默认地址，同一用户仅一个为 true |
| `createTime` | number | 否 | now | 创建时间戳 |

### 权限设置（控制台）
**所有用户不可读写**（即"仅管理端/云函数可访问"）。
> 原因：本项目前端不直连数据库，一律走云函数。云函数以管理员身份访问，不受此权限限制；前端无法越权读到他人数据。用户数据隔离由云函数内 `where({ _openid })` 保证，**不依赖**数据库的"仅创建者"权限。

## 3. 集合 `orders`（回收订单 · 核心）

| 字段 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| `_id` | string | 自动 | - | 主键 |
| `_openid` | string | **云函数手动写入** | - | 下单用户，createOrder 时由云函数 `getWXContext().OPENID` 赋值 |
| `orderNo` | string | 是 | - | 业务单号，云函数生成（日期+随机），仅展示 |
| `source` | string | 是 | - | `category` 按品类 / `photo` 拍照提交 |
| `summary` | string | **云函数生成** | - | 列表摘要文案，createOrder 时拼好落库（规则见接口契约 4.1），列表直接取用 |
| `items` | array | source=category 必填 | [] | 见 3.1 子结构 |
| `photos` | array<string> | source=photo 必填 | [] | 物品照片 fileID 列表 |
| `addressSnapshot` | object | 是 | - | 下单时地址快照，见 3.2。**存快照不存 addressId**，防止地址被改影响历史订单 |
| `appointDate` | string | 是 | - | 预约日期，如"2026-06-25" |
| `appointSlot` | string | 是 | - | 预约时间段，如"09:00-12:00" |
| `remark` | string | 否 | "" | 用户备注 |
| `status` | string | 是 | submitted | 订单状态，见 3.3 状态机 |
| `estimatePrice` | number | 否 | null | 平台估价（元） |
| `finalWeight` | number | 否 | null | 回收员反馈实际重量 |
| `finalPrice` | number | 否 | null | 实际成交金额（元） |
| `recyclerId` | string | 否 | null | 分配的回收员标识 |
| `createTime` | number | 是 | now | 下单时间戳 |
| `updateTime` | number | 是 | now | 最后更新时间戳 |

### 3.1 `items` 子结构
```jsonc
{ "categoryId":"c1", "categoryName":"纸箱/废纸", "estWeight":5, "estCount":0 }
```
| 字段 | 类型 | 说明 |
|---|---|---|
| `categoryId` | string | 品类 _id |
| `categoryName` | string | 冗余存品类名，避免详情页再查 categories |
| `estWeight` | number | 预估重量（unit=kg 时用） |
| `estCount` | number | 预估件数（unit=件 时用） |

### 3.2 `addressSnapshot` 子结构
```jsonc
{ "contactName":"张三", "phone":"138...", "region":"上海市浦东新区", "detail":"xx路1号" }
```
（字段同 addresses，去掉 isDefault/createTime）

### 3.3 状态机 `status`
```
submitted   已提交（待平台确认）   ← createOrder 初始状态
confirmed   平台已确认/已估价       ← 可填 estimatePrice
assigned    已分配回收员            ← 可填 recyclerId
recycling   上门回收中
completed   已完成                 ← 必填 finalWeight/finalPrice
canceled    已取消（用户取消）
rejected    不可回收（拍照件被拒）   ← 建议配 remark 说明原因
```

**流转规则：**
- 用户侧只能：`submitted/confirmed` → `canceled`（取消）。
- 其余流转均由管理端推进（控制台或 `adminUpdateOrder`）。
- `completed` / `canceled` / `rejected` 为终态，不可再变。

**状态 → 用户界面文案/进度（待确认）：**
| status | 展示文案 | 进度条 |
|---|---|---|
| submitted | 已提交，等待确认 | 1/4 |
| confirmed | 已确认，等待上门 | 2/4 |
| assigned / recycling | 回收员处理中 | 3/4 |
| completed | 已完成 | 4/4 |
| canceled | 已取消 | — |
| rejected | 暂不可回收 | — |

### 权限设置（控制台）
**所有用户不可读写**（即"仅管理端/云函数可访问"）。
> 同 addresses：前端不直连，全部走云函数；用户数据隔离由云函数内 `where({ _openid })` 保证。管理端用云函数以管理员身份操作，不受此限。

## 4. 云存储路径约定

| 用途 | 路径规则 | 说明 |
|---|---|---|
| 订单物品照片 | `recycle/{openid}/{timestamp}.jpg` | 前端 uploadFile 时指定 cloudPath |
| 品类图标 | `category/{name}.png` | 管理上传 |

数据库只存 uploadFile 返回的 `fileID`，展示时换临时 URL。

| 集合 | 索引字段 | 用途 |
|---|---|---|
| `orders` | `_openid` + `createTime`(降序) | 我的订单列表查询 |
| `orders` | `status` | 管理端按状态筛选 |
| `addresses` | `_openid` | 我的地址查询 |

> 配套：[云回收项目PRD.md](./云回收项目PRD.md) · [云回收技术路线.md](./云回收技术路线.md) · [数据字典.md](./数据字典.md) · [页面与交互流程.md](./页面与交互流程.md)
> 对应 Java 开发者认知：本文 ≈ API 接口文档（Swagger）。每个「接口」就是一个 `type`。
> 标注「待确认」的为依赖 PRD 缺口的默认设定，后续可改。

## 0. 通用约定

### 0.1 调用方式
前端统一通过云函数 `quickstartFunctions` 调用，用 `data.type` 指定动作：
```js
wx.cloud.callFunction({
  name: "quickstartFunctions",
  data: { type: "createOrder", /* 其它入参 */ },
});
```

### 0.2 统一返回结构
所有 type 返回如下结构（沿用现有云函数风格）：
```jsonc
{
  "success": true,        // 是否成功
  "data": { },            // 成功时的业务数据（对象或数组）
  "errMsg": ""            // 失败时的错误信息；success=true 时无此字段
}
```
失败示例：
```jsonc
{ "success": false, "errMsg": "ORDER_NOT_FOUND" }
```

### 0.3 身份与权限
- 无需传 userId / token。云函数内部用 `cloud.getWXContext().OPENID` 获取当前用户。
- 凡涉及"我的数据"（地址、订单）的接口，**只操作当前 openid 的数据**，前端无法越权。
- **⚠️ 写入时必须手动塞 `_openid`**：云函数是管理员身份，不会自动写入 openid。`createOrder`/`saveAddress` 等写接口须 `data._openid = OPENID`；查询接口须 `where({ _openid: OPENID })`。集合权限已收紧为前端不可直连（见数据字典 0 节）。

### 0.4 错误码（errMsg 取值）
| 错误码 | 含义 |
|---|---|
| `PARAM_INVALID` | 入参缺失或非法 |
| `ADDRESS_NOT_FOUND` | 地址不存在或非本人 |
| `ORDER_NOT_FOUND` | 订单不存在或非本人 |
| `ORDER_STATUS_INVALID` | 当前订单状态不允许该操作（如已完成的订单不能取消） |
| `BELOW_MIN_QUANTITY` | 按品类下单未达最低起收量 |
| `NO_PERMISSION` | 无权限（如非管理员调用管理接口） |
| `DB_ERROR` | 数据库异常 |

## 4. 订单 orders（核心）

### 4.1 `createOrder` — 提交回收订单
| 入参字段 | 必填 | 说明 |
|---|---|---|
| `source` | 是 | `"category"` 按品类 / `"photo"` 拍照提交 |
| `items` | source=category 时必填 | 数组：`[{ categoryId, categoryName, estWeight, estCount }]`（4 字段，按品类 unit 填 estWeight 或 estCount，另一个传 0） |
| `photos` | source=photo 时必填 | fileID 数组，**前端先 uploadFile 拿到 fileID 再传**，最多 9 张 |
| `addressId` | 是 | 选中的地址 _id，后端转成快照存入订单 |
| `appointDate` | 是 | 预约日期，格式 `"2026-06-25"` |
| `appointSlot` | 是 | 预约时间段，格式 `"09:00-12:00"` |
| `remark` | 否 | 备注 |

后端处理：校验地址归属 → **source=category 时校验最低起收量（见下）** → 生成 `orderNo` → 写 `addressSnapshot`、手动写 `_openid` → **生成 `summary` 落库**（见下）→ `status="submitted"` → 写入 createTime。

**最低起收量校验（仅 source=category）：** 汇总 items 的 estWeight/estCount，低于平台阈值（默认总重量 ≥ 5kg，阈值 Q2 待平台确认）时返回 `BELOW_MIN_QUANTITY`。source=photo 不校验。

**summary 生成规则（R6）：** 由 createOrder **在云函数内拼好并落库**到 `orders.summary`，列表接口直接返回，避免 getOrderList 反查 categories。规则：
- source=category：`"{首个品类名} 约{首项重量}{unit} 等{N}类"`，单品类时省略"等N类"，如 `"纸箱/废纸 约5kg"`。
- source=photo：`"拍照提交 {图片数}张"`，如 `"拍照提交 3张"`。

| 出参 | `{ success:true, data:{ _id, orderNo } }` |
| 失败 | `PARAM_INVALID` / `ADDRESS_NOT_FOUND` / `BELOW_MIN_QUANTITY` |

### 4.2 `getOrderList` — 我的订单列表
| 入参 | 说明 |
|---|---|
| `statusGroup` | 选填，对应列表 tab：`all`(默认/不传) / `ongoing`(进行中=submitted,confirmed,assigned,recycling) / `done`(已结束=completed,canceled,rejected) |
| `page` | 选填，页码，从 1 开始，默认 1 |
| `pageSize` | 选填，每页条数，默认 20，最大 50 |
| 出参 data | `{ list, total, hasMore }`，list 为订单精简字段数组，按 createTime 倒序 |
```jsonc
{ "success": true, "data": {
  "list": [
    { "_id":"o1", "orderNo":"20260622001", "source":"category", "status":"submitted",
      "summary":"纸箱/废纸 约5kg 等2类", "appointDate":"2026-06-25", "appointSlot":"09:00-12:00", "createTime":169... }
  ],
  "total": 37, "hasMore": true
} }
```

### 4.3 `getOrderDetail` — 订单详情
| 入参 | `id`（订单 _id） |
|---|---|
| 后端处理 | 校验归属；把 `photos` 的 fileID 批量换成临时 URL 一并返回 |
| 出参 data | 完整订单 + `photoUrls` |
```jsonc
{ "success": true, "data": {
  "_id":"o1","orderNo":"20260622001","source":"category","status":"confirmed",
  "items":[{"categoryId":"c1","categoryName":"纸箱/废纸","estWeight":5,"estCount":0}],
  "photos":["cloud://..."], "photoUrls":["https://临时url"],
  "addressSnapshot":{"contactName":"张三","phone":"138...","region":"上海...","detail":"xx路1号"},
  "appointDate":"2026-06-25","appointSlot":"09:00-12:00","remark":"",
  "estimatePrice":4, "finalWeight":null, "finalPrice":null,
  "createTime":169..., "updateTime":169...
} }
```
| 失败 | `ORDER_NOT_FOUND` |

### 4.4 `cancelOrder` — 用户取消订单
| 入参 | `id` |
|---|---|
| 规则 | 仅 `submitted` / `confirmed` 状态可取消（上门前），其它返回 `ORDER_STATUS_INVALID` |
| 出参 | `{ success:true }` |

### 4.5 `getTempFileURL` — 图片 fileID 换临时链接（可选）
| 入参 | `fileIDs`（数组） |
|---|---|
| 出参 | `{ success:true, data:[{ fileID, tempFileURL }] }` |
| 备注 | 前端也可直接用 `wx.cloud.getTempFileURL`，本接口供需要后端统一处理时用 |
